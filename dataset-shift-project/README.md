# Dataset Shift: How Classical ML Models Respond to Distribution Changes

**A comparative study of 7 classical machine learning models evaluating their robustness against various types of dataset shift.**

## Motivation
Real-world machine learning models often fail when deployed because the data they encounter in production differs from the data they were trained on (dataset shift). Understanding how and why different classical ML algorithms degrade under specific distribution changes is critical for building more robust, reliable systems—especially when deep learning is not an option due to computational or interpretability constraints.

## Dataset Details
- **Name**: [Adult Income Dataset](https://archive.ics.uci.edu/ml/datasets/adult) (OpenML ID 1590)
- **Source**: UCI Machine Learning Repository
- **Size**: ~48,842 instances
- **Features**: 14 attributes (categorical and continuous features such as age, workclass, education, marital-status, occupation, etc.)
- **Target**: Binary classification (predicting whether income >$50K/yr or not)

## Models Studied
We evaluate the following 7 classical models using `scikit-learn`:
1. Naïve Bayes
2. Logistic Regression
3. Support Vector Machine (RBF kernel)
4. Decision Tree
5. Random Forest
6. Gradient Boosting
7. AdaBoost

## Shift Types Investigated
All models are trained on a fixed training set. To simulate real-world degradation, the following shifts are applied **only to the test set**:
1. **Covariate Shift**: The distribution of features changes, but the relationship between features and the target remains the same (simulated via Gaussian noise and feature scaling drift).
2. **Prior Probability Shift**: The distribution of target classes changes over time (simulated via class resampling to alter the base rates).
3. **Concept-Adjacent Shift**: The underlying meaning or quality of the most informative features deteriorates (simulated via deliberate corruption or masking of highly predictive features).

## Installation & Usage

1. **Clone the repository:**
   ```bash
   git clone <your-repo-url>
   cd dataset-shift-project
   ```

2. **Create a virtual environment and install dependencies:**
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows, use `venv\Scripts\activate`
   pip install -r requirements.txt
   ```

3. **Run the experiments:**
   This script trains the models, applies shifts across varying intensities, computes evaluation metrics, and saves the data to `results/experiment_results.csv`.
   *Note: Training the SVM model may take a few minutes!*
   ```bash
   python src/run_experiments.py
   ```

4. **Launch the Streamlit App:**
   Interact with the trained results, visualize performance curves, and compare models side-by-side.
   ```bash
   streamlit run app.py
   ```

## Expected Outputs
- **Plots & Visualizations**: Beautiful Seaborn line charts showing performance degradation (Accuracy, F1, ROC-AUC, Brier Score) across shift intensities.
- **Heatmaps**: Comparative matrices showing which models are most resilient to specific shift types across all parameters.
- **Streamlit App**: An interactive dashboard showing live results loaded securely via CSV caching.
